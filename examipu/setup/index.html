ExamForge セットアップウィザードSC-20

ExamForge をあなたのGitHub＋GASに展開する

このウィザードは、あなたのGitHubアカウントにExamForge一式を配置し、Google Apps Script（GAS） バックエンドと接続するための初回セットアップ専用ツールです。導入手順書v1.1§5・§6の手順に沿って 進みます。所要時間の目安は合計10分程度です。

通信先について: このウィザードのみ GitHub REST API（api.github.com）と通信します。 受験・教員アプリ本体は github.io（自オリジン）と GAS、Google Identity Services 以外とは 一切通信しません（NFR-01）。ウィザードは配布物ではなくセットアップ時にのみ使うツールです。

① GitHubリポジトリ情報の入力

⚠️ このウィザードはリポジトリを新規作成しません。 下のリポジトリ名は、GitHubの画面上であなたが既に作成済みのリポジトリの名前を入力してください （PATの仕組み上、存在しないリポジトリは指定できません）。まだリポジトリが無い場合は、 先に github.com/new で 空のリポジトリを1つ作ってから、この画面に戻ってきてください（導入手順書§3・§5参照）。

Fine-grained Personal Access Token（github_pat_...）を使用します。 「Repository access: Only select repositories」で上で作成したリポジトリを選択し、 「Permissions」は Contents: Read and write と Pages: Read and write のみを 付与してください（導入手順書§3・外部設計書R-07）。このトークンはこのページのメモリ内にのみ 保持され、保存やどこかへの送信（GitHub API以外）は行いません。

入力例
・ExamForge専用の新しいリポジトリを作った場合 → リポジトリ名だけを入力する
　　（例：GitHubで examforge-2027 という名前のリポジトリを作った → examforge-2027 と入力）
・すでに <GitHubユーザー名>.github.io という名前でホームページを公開している場合
　　→ そのリポジトリ名をそのまま入力する
　　（例：ユーザー名が yamada-taro なら、リポジトリ名は yamada-taro.github.io と入力。 　　「your-username」の部分は自分のユーザー名に置き換える点に注意）
・上のどちらの場合も、リポジトリ名の欄には https:// や github.com/ は含めない

既存のホームページ用リポジトリ（<GitHubユーザー名>.github.io）に相乗りする場合は、 ここに新しいフォルダ名を入力してください（例：examforge-2027）。ExamForge一式はこの フォルダの中にだけ配置され、既存のホームページのファイルには一切触れません。ExamForge専用の 新しいリポジトリを作った場合は、空欄のままで構いません（リポジトリ直下に配置されます）。

② 自動展開中

リポジトリの確認→静的ファイル一括配置→GitHub Pages有効化を自動で行います。 GitHub Pagesの反映には数十秒〜数分かかることがあります（気長にお待ちください）。

③ GAS（Google Apps Script）へのコード貼付・デプロイ

  1. 個人のGoogleアカウントでログインした状態で script.google.com を開き、「新しいプロジェクト」を作成する（画面右上のアイコンで大学アカウントでログインしたままになっていないか確認）。
  2. デフォルトのサンプルコードを全て削除し、下のテキストエリアの内容を貼り付ける。
  3. 「デプロイ」→「新しいデプロイ」→種類「ウェブアプリ」を選び、実行するユーザー：自分／アクセスできるユーザー：全員、でデプロイする。
  4. 初回の権限確認画面が出たら「詳細」→「（プロジェクト名）に移動」→「許可」を選ぶ。
  5. 発行された .../exec で終わるURLをコピーし、次のステップで貼り戻す。
